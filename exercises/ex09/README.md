# Exercise 9: Arrays and Strings

## 学习目标

- 理解数组的部分初始化
- 理解C字符串以`\0`结尾
- 区分字符数组和字符串指针
- 观察未初始化数组和缺少结束符的后果
- 理解char和int之间的关系

## 部分初始化

    int numbers[4] = {0};

第一个元素初始化为0，剩余元素也会自动初始化为0。

    char name[4] = {'a'};

实际内容为：

    'a', '\0', '\0', '\0'

因此它是一个合法的C字符串。

## 手动构造字符串

    name[0] = 'Z';
    name[1] = 'e';
    name[2] = 'd';
    name[3] = '\0';

最终字符串为`"Zed"`。

## 字符数组与指针

    char name[] = "Zed";

创建可修改的字符数组，`sizeof(name)`为4。

    const char *another = "Zed";

创建指向字符串字面量的指针。在64位系统中，`sizeof(another)`通常为8，也就是指针大小。

字符串字面量不应该被修改。

## 缺少结束符

如果字符数组中没有`\0`，`printf("%s")`会继续读取数组之外的内存。

程序可能输出乱码、看似正常或发生崩溃，这属于未定义行为。

Valgrind可能报告：

    Invalid read of size 1

## 字符和整数

字符本质上是小整数。

将name的元素用`%d`打印，可能得到：

    90 101 100 0

分别对应：

    'Z', 'e', 'd', '\0'

将字符赋给int数组，再使用`%c`打印，通常不会产生编译警告，因为`%c`接收int参数。

## 多种输出方法

- `%s`：输出到`\0`
- `%c`：逐字符输出
- `puts`：输出字符串并自动换行
- `putchar`：输出单个字符
- `fwrite`：按照明确的字节数量输出

## 将字符字节解释为整数

不能安全地直接把char数组强制转换为int指针，因为可能存在对齐、严格别名和字节序问题。

更安全的实验方式是使用`memcpy`复制到`uint32_t`中。

不同CPU字节序可能导致不同的整数结果。

## Valgrind结果

最终正确版本应显示：

    ERROR SUMMARY: 0 errors from 0 contexts

## 本题总结

C字符串并不是独立的数据类型，而是以`\0`结尾的char数组。

字符数组和指针虽然都能传给`%s`，但它们的大小、存储方式和可修改性不同。
