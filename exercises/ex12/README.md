# Exercise 12: If, Else-If, Else

## 学习目标

- 学习if、else if和else
- 理解比较表达式
- 使用&&、||和!
- 理解短路求值
- 学习break提前退出循环
- 区分内存错误和逻辑错误

## 条件分支

基本结构：

    if (条件一)
    {
        代码一;
    }
    else if (条件二)
    {
        代码二;
    }
    else
    {
        代码三;
    }

条件会从上到下依次检查，最多只有一个分支会执行。

## argc与用户参数

`argc`包含程序名称。

执行：

    ./ex12 one two

通常得到：

    argc = 3
    argv[0] = "./ex12"
    argv[1] = "one"
    argv[2] = "two"

因此用户参数数量为：

    argc - 1

当`argc == 1`时，表示用户没有提供额外参数。

## 比较运算符

- `==`：等于
- `!=`：不等于
- `<`：小于
- `<=`：小于等于
- `>`：大于
- `>=`：大于等于

`=`是赋值，`==`才是相等比较。

## 逻辑运算符

- `&&`：逻辑与，两边都为真才为真
- `||`：逻辑或，至少一边为真就为真
- `!`：逻辑非，反转真假

原程序条件：

    argc > 1 && argc < 4

表示argc必须同时大于1并且小于4。

## 短路求值

`&&`左边为假时，右边不会继续计算。

`||`左边为真时，右边不会继续计算。

短路求值可以用于保护数组和指针访问。

## 将&&改成||

改成：

    argc > 1 || argc < 4

后，只要argc大于1，中间分支就会成立，导致最后的else无法执行。
属于逻辑错误，不一定产生编译警告或Valgrind错误。

## 删除else

删除else后，参数过多时程序没有任何提示，但仍可能正常结束。

else用于处理前面所有条件都不成立的情况。

## break

`break`会立即退出最近的一层循环。

    if (i == 2)
    {
        break;
    }

循环结束后，程序仍会继续执行后面的代码。

`return`则会结束整个当前函数。

## Valgrind结果

最终正确程序应显示：

    ERROR SUMMARY: 0 errors from 0 contexts

Valgrind主要检查内存错误，不能代替逻辑测试。

## 本题总结

条件分支必须正确处理所有边界情况。

`&&`与`||`含义完全不同，替换后可能让某些分支永远无法执行。

编译无警告和Valgrind无错误，并不代表程序逻辑一定正确，还需要设计不同输入进行测试。
