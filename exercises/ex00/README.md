# Exercise 0: The Setup

## 学习目标

- 在Linux中配置C语言开发环境
- 熟悉终端和Shell提示符
- 安装并验证GCC、Make、GDB和Valgrind
- 学会使用终端文本编辑器
- 使用Git保存学习过程

## 当前环境

本练习在Ubuntu Linux环境中完成。

## 已安装工具

- GCC：用于编译C程序
- Make：用于自动化构建
- GDB：用于程序调试
- Valgrind：用于检查内存错误
- Vim和Nano：用于编辑文本
- Git：用于版本管理

## 对“不要使用IDE”的理解

书中并不是说IDE永远不能使用，而是强调初学阶段需要理解代码背后的编译、链接、运行和调试过程。

如果完全依赖IDE中的按钮，就可能不知道实际执行了哪些编译命令，也不容易理解错误发生在哪个阶段。

因此，本阶段优先使用终端、文本编辑器、GCC、Make和Git完成练习。

## 本题总结

练习0没有正式的C语言代码，主要目标是搭建开发环境，并为后面的练习做好准备。
