## 学习目标

- 理解C程序的基本组成
- 认识预处理指令、函数、变量和注释
- 初步理解main函数的命令行参数
- 理解语句、代码块和返回值
- 使用Valgrind验证程序

## 程序结构

一个简单的C程序通常包括：

1. 头文件
2. 注释
3. main函数
4. 变量声明和初始化
5. 函数调用
6. return语句

## 关键符号

### `#`

表示预处理指令的开始。

### `< >`

在`#include <stdio.h>`中表示从系统头文件目录查找头文件。

### `( )`

可用于函数参数列表、函数调用和表达式分组。

### `{ }`

表示一个代码块的开始和结束。

### `;`

表示一条普通C语句结束。

### `*`

在`char *argv[]`中表示指针。这里表示数组元素是指向字符的指针，也就是字符串。

### `[ ]`

表示数组。`argv[]`表示一个数组参数。

### `=`

赋值或初始化，把右侧的值放入左侧变量。

### `,`

分隔函数参数。

### `%d`

printf格式占位符，表示输出一个int类型的十进制整数。

### `\n`

换行转义序列。

### `/* */`

多行注释。

### `//`

单行注释。

## main函数

    int main(int argc, char *argv[])

- `argc`表示命令行参数数量
- `argv`表示命令行参数字符串数组
- `argv[0]`通常是程序自己的名称
- `main`返回int类型的退出状态

本程序暂时没有使用`argc`和`argv`，因此使用：

    (void)argc;
    (void)argv;

明确告诉编译器这些参数是有意未使用的。

## 变量

    int distance = 100;

这行同时完成：

- 声明一个int变量
- 将变量命名为distance
- 使用100初始化它

## printf调用

    printf("You are %d miles away.\n", distance);

`distance`的值会替换格式字符串中的`%d`。

## 返回值

    return 0;

表示程序正常结束，并向操作系统返回成功状态。

## Valgrind结果

程序在Valgrind中显示：

## 补充

知道了Valgrind 的 --track-origins=yes 的使用方法以及具体原因，如下：

它会在你的程序刚启动时，给每一个分配出来的内存（每一张发出去的身份证）都偷偷安上一个微型 GPS 追踪器。

### 但是！！！

既然这么好用，为什么不默认开启？

答案是：极其消耗性能。

给内存里的每一个字节都挂上“GPS 追踪器”并时刻记录它们的流向，是一件非常繁重的工作。

开启 --track-origins=yes 会导致你的程序运行速度慢上 2 到 3 倍，同时消耗更多的内存。
